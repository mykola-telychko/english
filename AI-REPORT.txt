Оцінка проекту (швидкий огляд)

Плюси
- проста структура, чисті HTML-файли, легко читати.
- є навігація / тематичні сторінки, прийнятний контент для навчання.
- є CSS окремо, ймовірно, стиль працює.

Недоліки
- жорсткі HTML сторінки без шаблонів → дублювання (header/footer/меню повторюється).
- не використовуються семантичні атрибути ARIA, accessibility.
- немає мінімальної SEO оптимізації (meta description, keywords, title variation).
- виключно статичний без CMS або скриптів, складно масштабувати.
- у файлі d11-reich.html є HTML-помилка: вкінці діалогу пропущено `</p>` закривальний тег в останньому `<div class="dialogue">`.
- без розділення змісту / презентації, затруднює підтримку (текст + структура в одному місці).
- немає автоматичних тестів, лінтингу, CICD.
- назви файлів неінтуїтивні (d11-*, 1-ussp etc) — ускладнює супровід.

План виправлення
1. Переход на шаблони:
   - зібрати header/footer/nav в один фрагмент (SSI, Jekyll, eleventy, простий JS).
   - зменшити повторення.
2. Виправити HTML:
   - перевірити валідність через W3C validator.
   - виправити незакриті теги.
3. Контент:
   - `meta description` + `og:` для кожної сторінки.
   - зробити заголовки й структуру спрощено.
4. Доступність:
   - додати `lang`, ARIA roles, `alt` для зображень.
   - перевірити контраст, клавіатурну навігацію.
5. Можливість масштабування:
   - Якщо плануєш додати 50+ уроків — переходь на статичний генератор (Hugo/Eleventy), або технічне рішення з JSON/Markdown.
6. CI/QA:
   - додати простий скрипт `npm test` / `htmlhint` / `prettier` / `stylelint`.
   - настройка GitHub Actions (build + validate).
7. Структура:
   - перейменувати файли в читаємі `third-reich.html`, `economics.html`, `history/third-reich.html`.
   - у `README` описати правила редагування.
8. Безпека:
   - перевірити відсутність inline JS/небезпечної вставки, якщо будете додавати форму.
9. Контентний фокус:
   - ділити дуже довгі діалоги на кілька частин (клас “practice”, “debate”).
   - зробити міжнародну локалізацію (i18n) якщо треба.
